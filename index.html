Automatiseret datafangst: fra sagsmapper til CRM

Forside (trin-for-trin)

Hvorfor (WHY)

For at få et brugbart CRM og et kommercielt overblik uden workshops og uden at partnerne skal bruge tid på manuel indtastning, kopiering eller “jagt” på oplysninger.

Hvad (WHAT)

Systemet henter data fra eksisterende kilder (fx faktura, sagsmapper og mails), strukturerer det og opretter CRM-udkast (konto, kontakt, mulighed og næste skridt) som kan godkendes med ét klik.

1) Hent

Indsamler signaler og metadata fra kilder

2) Strukturér

Matcher virksomhed/kontakt og rydder dubletter

3) Opret udkast

CRM-udkast → godkend med 1 klik

Typisk frigjort tid

8–12t

pr. uge pr. partner (demo-estimat)

Sådan læses denne forside

Forsiden forklarer input og flow: hvordan data bliver til CRM-udkast. De næste trin viser Python-scriptet (bevis for automatisk udtræk) og CRM-populating (hvordan udkast godkendes). Først derefter giver “Kommercielt overblik” og “Kundeudvikling” mening.

Trin 1

Vi henter signaler

Systemet kobler sig på kendte kilder og henter ændringer og mønstre. Ikke lange dokumenter – men “hvad er nyt?” og “hvad stikker ud?”.

Teknisk note

Baseret på automatiserede scripts (Python), der parser sagsmapper og fakturaer og udleder relevante kommercielle triggers (fx ændret aktivitet, nye nøglepersoner, skift i praksisområde).

Sager Tid Fakturering Sagsmapper Mails (headers) Offentlige registre

Trin 2

Vi gør det forståeligt

Hver anbefaling får en kort forklaring: kilder, opdateringsdato, dækning og hvad der mangler. Det gør det trygt at handle – og trygt at lade systemet overvåge videre.

Eksempel

“Ny beslutningstager + ændret aktivitetsmønster” → foreslået opfølgning + CRM-udkast.

Trin 3

Vi opretter CRM-udkast

Når du trykker “Godkend”, oprettes alt i CRM: konto, kontakt, mulighed, opgaver og opfølgning. Hvis du vælger Overvågning, logges der stadig – uden at “fylde” i pipelinen.

Ingen manuel indtastning

Systemet gør forarbejdet. Partneren tager kun beslutningen.

Python-scriptet (kører i baggrunden)

Bevis: ingen manuel partner-input

Hvorfor (WHY)

For at få et CRM der faktisk bliver fyldt med rigtige data, uden at partnerne skal ændre adfærd.

Hvad (WHAT)

Scriptet udtrækker virksomhed, kontakt, relation og signaler fra fakturaer, sagsmapper og mails og bygger CRM-udkast.

Manuel indsats

0 min

for partneren

Hvad finder scriptet?

Entiteter: virksomhed, kontakt, rolle. Relationer: partner-ejer, team, historik. Signaler: ændret købsmønster, nye beslutningstagere, stigende aktivitet i et område.

Flow

Extraction pipeline

  • Læs faktura/PDF → udtræk CVR, firmanavn, beløb, praksisområde
  • Scan sagsmapper → udtræk modparter, stakeholders, type
  • Scan mails (headers/metadata) → udtræk kontakter + relationstæthed
  • Normalisér → deduplikér → match til CRM-konti
  • Byg udkast: konto + kontakt + mulighed + næste skridt

Eksempel (illustrativ)

Python snippet

# 1) Ingest
invoices = load_pdfs("/invoices/")
cases    = load_case_folders("/cases/")
emails   = load_mail_headers("/mail/")  # metadata only

# 2) Extract entities
entities = extract_entities(invoices, cases, emails)  # company, people, roles

# 3) Resolve + dedupe
resolved = resolve_identities(entities)  # CVR match, name match, email match

# 4) Signals
signals = compute_signals(resolved)  # spend up/down, new decision maker, practice shift

# 5) Produce CRM drafts (NOT live)
drafts = build_crm_drafts(resolved, signals)

# 6) Push drafts to CRM for 1-click approval
push_drafts_to_crm(drafts)

Pointen er ikke koden i sig selv — men at den kører automatisk og producerer et struktureret “udkast”.

CRM-populating (udkast → godkend)

CRM købes udefra • vi fylder det

Hvorfor (WHY)

For at CRM ikke ender som en “tom skal”. Data skal lande som brugbare udkast, ellers dør adoptionen.

Hvad (WHAT)

Output fra datafangst/Python lander som klare udkast i CRM: konto, kontakt, mulighed og næste skridt. Partneren godkender — systemet opretter alt uden manuel indtastning.
